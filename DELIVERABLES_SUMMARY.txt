SEGMENTATION COMPARISON DETAILS
================================================================================

WITHOUT POST-PROCESSING SEGMENTATION
-------------------------------------
Method:
  • Raw neural network predictions
  • Sigmoid activation function
  • Fixed threshold at 0.5
  • No morphological operations
  • No object size filtering

Characteristics:
  • Faster inference time
  • More false positives
  • Noisier masks
  • Higher recall (fewer missed spills)
  • Lower precision (more false alarms)

Use Case:
  • Research and development
  • Model architecture comparison
  • Speed-critical applications


WITH POST-PROCESSING SEGMENTATION
----------------------------------
Method:
  • Adaptive thresholding (model-specific)
  • Morphological opening (noise removal)
  • Morphological closing (hole filling)
  • Small object removal (filtering)
  • Spatial coherence enforcement

Model-Specific Parameters:

  SegNet:
    • Threshold: 60th percentile (min 0.45)
    • Opening kernel: 5 pixels
    • Closing kernel: 7 pixels
    • Min object size: 200 pixels
    • Strategy: Conservative, high precision

  DeepLabV3+:
    • Threshold: 95th percentile or Otsu (max 0.95)
    • Opening kernel: 3 pixels
    • Closing kernel: 5 pixels
    • Min object size: 100 pixels
    • Strategy: Aggressive, high recall

  Hybrid:
    • Threshold: 92nd percentile (0.65-0.92)
    • Opening kernel: 3 pixels
    • Closing kernel: 6 pixels
    • Min object size: 120 pixels
    • Strategy: Balanced, optimal F1-score

Performance Impact:
  ✓ +5-10% precision improvement
  ✓ +2-5% ROC-AUC improvement
  ✓ +3-7% specificity improvement
  ✓ Significantly cleaner masks
  ✗ ~1-3% recall reduction
  ✗ 2-3x slower inference


ROC CURVE ANALYSIS
================================================================================

What ROC Curves Show:
  • Trade-off between True Positive Rate and False Positive Rate
  • Performance across all possible thresholds
  • Overall discriminative ability of the model

Interpretation:
  • Curve closer to top-left = better performance
  • Area Under Curve (AUC) = overall metric
  • Diagonal line = random guessing (AUC = 0.5)

AUC Value Guide:
  • 0.90-1.00: Excellent discrimination
  • 0.80-0.90: Good discrimination
  • 0.70-0.80: Fair discrimination
  • 0.60-0.70: Poor discrimination
  • 0.50-0.60: No discrimination (random)

Expected Results:
  WITHOUT Segmentation:
    • SegNet AUC: 0.85-0.90
    • DeepLabV3+ AUC: 0.88-0.93
    • Hybrid AUC: 0.90-0.95

  WITH Segmentation:
    • SegNet AUC: 0.87-0.92
    • DeepLabV3+ AUC: 0.90-0.94
    • Hybrid AUC: 0.92-0.96


CONFUSION MATRIX ANALYSIS
================================================================================

Matrix Structure:
                    Predicted Label
                 No Oil  |  Oil Spill
    Actual   No Oil   TN  |     FP      ← Specificity = TN/(TN+FP)
    Label  Oil Spill  FN  |     TP      ← Recall = TP/(TP+FN)
                           ↑        ↑
                    Precision = TP/(TP+FP)

Key Insights:

True Negatives (TN) - Top Left:
  • Correctly identified non-oil spill pixels
  • Should be LARGEST number
  • High TN = good specificity

False Positives (FP) - Top Right:
  • Incorrectly predicted as oil spill
  • Should be MINIMIZED
  • High FP = too many false alarms
  • Post-processing reduces FP significantly

False Negatives (FN) - Bottom Left:
  • Missed oil spill pixels
  • Should be MINIMIZED
  • High FN = missing actual spills
  • Critical for emergency response

True Positives (TP) - Bottom Right:
  • Correctly identified oil spill pixels
  • Should be MAXIMIZED
  • High TP = good detection

Expected Patterns:
  WITHOUT Segmentation:
    • Higher TP (good detection)
    • Higher FP (more false alarms)
    • Lower TN (less specific)

  WITH Segmentation:
    • Similar or slightly lower TP
    • Much lower FP (fewer false alarms)
    • Higher TN (more specific)
    • Better overall balance


METRICS EXPLANATION
================================================================================

1. ACCURACY = (TP + TN) / (TP + TN + FP + FN)
   What it means: Overall percentage of correct predictions
   Range: 0 to 1 (higher is better)
   Note: Can be misleading with imbalanced datasets

2. PRECISION = TP / (TP + FP)
   What it means: Of all predicted oil spills, how many are correct?
   Range: 0 to 1 (higher is better)
   Important when: False alarms are costly

3. RECALL = TP / (TP + FN)
   What it means: Of all actual oil spills, how many did we detect?
   Range: 0 to 1 (higher is better)
   Important when: Missing spills is costly
   Also called: Sensitivity, True Positive Rate

4. F1-SCORE = 2 × (Precision × Recall) / (Precision + Recall)
   What it means: Balanced metric combining precision and recall
   Range: 0 to 1 (higher is better)
   Important: Best overall performance indicator

5. SPECIFICITY = TN / (TN + FP)
   What it means: Of all non-oil areas, how many are correctly identified?
   Range: 0 to 1 (higher is better)
   Important: Measures false alarm rate

6. ROC-AUC = Area Under ROC Curve
   What it means: Overall discriminative ability across all thresholds
   Range: 0 to 1 (higher is better)
   Important: Threshold-independent performance measure
