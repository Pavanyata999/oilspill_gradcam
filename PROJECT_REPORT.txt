PROJECT OVERVIEW
================================================================================

Title: Hybrid Deep Learning Approach for SAR-based Oil Spill Detection

Abstract:
This project presents a novel hybrid deep learning architecture that combines
SegNet and DeepLabV3+ for detecting oil spills in Synthetic Aperture Radar (SAR)
satellite imagery. The system leverages the complementary strengths of both
architectures to achieve robust and accurate oil spill detection. The hybrid
model demonstrates superior performance compared to individual baseline models
through an intelligent fusion mechanism that combines feature-level and
decision-level integration.

================================================================================
1. INTRODUCTION
================================================================================

1.1 Background
--------------
Oil spills pose severe environmental threats to marine ecosystems, affecting
wildlife, coastlines, and local economies. Rapid and accurate detection of oil
spills is crucial for timely response and mitigation efforts. SAR satellite
imagery provides all-weather, day-and-night monitoring capabilities, making it
ideal for marine surveillance applications.

1.2 Problem Statement
---------------------
Traditional oil spill detection methods face several challenges:
• Manual interpretation is time-consuming and subjective
• Look-alike phenomena (e.g., algae, seaweed) cause false positives
• Variable oil spill characteristics require adaptable detection systems
• Real-time processing demands efficient algorithms

1.3 Objectives
--------------
Primary objectives of this project:
1. Develop a hybrid deep learning model combining SegNet and DeepLabV3+
2. Achieve high accuracy in oil spill detection from SAR imagery
3. Compare performance with and without post-processing segmentation
4. Provide comprehensive evaluation metrics including ROC curves and confusion matrices
5. Create a production-ready system for operational use

2.1 Dataset

Source: PALSAR and Sentinel-1 SAR Satellites
Image Resolution: Variable (standardized to 512x512 for processing)
Total Images: Training + Test samples from PALSAR sensor

Structure:
• Training Set: Located in dataset/SOS_dataset/train/palsar/
  - Contains SAR images (*_sat.jpg) and corresponding masks (*_mask.png)
  - Augmented with horizontal/vertical flips, rotations, brightness adjustments

• Test Set: Located in dataset/SOS_dataset/test/palsar/
  - Satellite images in sat/ subdirectory
  - Ground truth masks in gt/ subdirectory
  - Used for final evaluation and metrics calculation

Image Characteristics:
• Grayscale SAR intensity images
• Binary masks (oil spill vs. seawater)
• Variable oil spill sizes and shapes
• Presence of speckle noise typical of SAR imagery

2.2 Model Architecture
----------------------

The hybrid model consists of three main components:

A) SegNet Architecture
   - Encoder-decoder structure with symmetric design
   - VGG-16 style encoder with batch normalization
   - Pooling indices stored for precise upsampling
   - Strengths: Precise boundary delineation, memory efficient
   - Characteristics: Conservative predictions, clean detections

B) DeepLabV3+ Architecture
   - ResNet-50 backbone for feature extraction
   - Atrous Spatial Pyramid Pooling (ASPP) module
   - Multi-scale feature aggregation
   - Decoder for detailed boundary recovery
   - Strengths: Multi-scale context, high coverage
   - Characteristics: Aggressive predictions, comprehensive coverage

C) Hybrid Fusion Architecture
   - Feature-level fusion from encoder layers
   - Multi-scale feature integration
   - Attention mechanisms for adaptive weighting
   - Decision-level fusion of segmentation outputs
   - Learnable fusion parameters
   - Strengths: Combines best of both models, balanced performance

Model Parameters:
• Total trainable parameters: Approximately 45-50 million
• Input size: 512x512x3 (RGB converted from grayscale)
• Output: 512x512x1 (binary segmentation mask)

2.3 Training Strategy
---------------------

Loss Functions:
1. Binary Cross-Entropy (BCE) Loss
   - Pixel-wise classification loss
   - Weight: 1.0

2. Dice Loss
   - Addresses class imbalance
   - Focuses on overlap between prediction and ground truth
   - Weight: 1.0

3. Focal Loss
   - Addresses hard example mining
   - Parameters: alpha=1, gamma=2
   - Weight: 1.0

Combined Loss = BCE + Dice + Focal

Optimization:
• Optimizer: Adam with weight decay
• Initial Learning Rate: 1e-4
• Learning Rate Schedule: ReduceLROnPlateau (factor=0.5, patience=5)
• Batch Size: 4-8 (depending on GPU memory)
• Epochs: 50-100 with early stopping (patience=10)

Data Augmentation:
• Horizontal flip (p=0.5)
• Vertical flip (p=0.5)
• Random rotation 90° (p=0.5)
• Random brightness/contrast (p=0.5)
• Normalization: ImageNet statistics

2.4 Post-Processing Segmentation
---------------------------------

The system implements two evaluation approaches:

A) WITHOUT Post-Processing
   - Raw model predictions with sigmoid activation
   - Simple threshold at 0.5
   - No morphological operations
   - Represents pure neural network performance

B) WITH Post-Processing (Advanced Segmentation)
   
   Model-Specific Thresholds:
   • SegNet: 60th percentile (min 0.45)
     - Most conservative approach
     - Reduces false positives
     
   • DeepLabV3+: 95th percentile or Otsu's method (max 0.95)
     - Most aggressive approach
     - Maximizes coverage
     
   • Hybrid: 92nd percentile (0.65-0.92 range)
     - Balanced approach
     - Optimal trade-off

   Morphological Operations:
   1. Opening (erosion + dilation)
      - Removes small noise and isolated pixels
      - Kernel size: 3-5 pixels (model-dependent)
      
   2. Closing (dilation + erosion)
      - Fills small holes within detections
      - Kernel size: 5-7 pixels (model-dependent)
      
   3. Small Object Removal
      - Removes connected components below threshold
      - Minimum sizes: 100-200 pixels (model-dependent)
   
   Benefits:
   • Reduces false positives significantly
   • Improves spatial coherence
   • Creates production-ready masks
   • Maintains true positive detections

4. EVALUATION METRICS
================================================================================

4.1 Metrics Overview
--------------------

The system evaluates model performance using six key metrics:

1. Accuracy
   Definition: (TP + TN) / (TP + TN + FP + FN)
   Interpretation: Overall correctness of predictions
   Range: 0 to 1 (higher is better)

2. Precision (Positive Predictive Value)
   Definition: TP / (TP + FP)
   Interpretation: What percentage of predicted oil spills are actual oil spills
   Range: 0 to 1 (higher is better)
   Importance: Critical for minimizing false alarms

3. Recall (Sensitivity, True Positive Rate)
   Definition: TP / (TP + FN)
   Interpretation: What percentage of actual oil spills are detected
   Range: 0 to 1 (higher is better)
   Importance: Critical for not missing oil spills

4. F1-Score
   Definition: 2 × (Precision × Recall) / (Precision + Recall)
   Interpretation: Harmonic mean of precision and recall
   Range: 0 to 1 (higher is better)
   Importance: Balanced metric for imbalanced datasets

5. Specificity (True Negative Rate)
   Definition: TN / (TN + FP)
   Interpretation: Ability to correctly identify non-oil spill areas
   Range: 0 to 1 (higher is better)

6. ROC-AUC (Area Under ROC Curve)
   Definition: Area under the Receiver Operating Characteristic curve
   Interpretation: Overall discriminative ability across all thresholds
   Range: 0 to 1 (higher is better, 0.5 = random)
   Importance: Threshold-independent performance measure

4.2 ROC Curves
--------------

ROC (Receiver Operating Characteristic) curves visualize the trade-off between
True Positive Rate (Recall) and False Positive Rate (1 - Specificity) across
different classification thresholds.

Interpretation:
• Curve closer to top-left corner = better performance
• Diagonal line represents random guessing (AUC = 0.5)
• AUC closer to 1.0 indicates excellent discrimination

Generated Files:
• roc_curves_comparison.png: Side-by-side comparison of ROC curves
  - Left panel: WITHOUT post-processing
  - Right panel: WITH post-processing
  - Shows all three models (SegNet, DeepLabV3+, Hybrid)
  - Includes AUC values in legend

Expected Results:

• DeepLabV3+ shows high TPR but may have higher FPR
• SegNet shows conservative performance with lower FPR

4.3 Confusion Matrices
----------------------

Confusion matrices provide detailed breakdown of classification results:

Structure:
                 Predicted
              No Oil | Oil Spill
Actual  No Oil    TN  |    FP
       Oil Spill  FN  |    TP

Where:
• TN (True Negative): Correctly identified non-oil spill pixels
• FP (False Positive): Incorrectly identified as oil spill
• FN (False Negative): Missed oil spill pixels
• TP (True Positive): Correctly identified oil spill pixels

Generated Files:
• confusion_matrices_without_segmentation.png
• confusion_matrices_with_segmentation.png

Each file contains three confusion matrices (SegNet, DeepLabV3+, Hybrid)

Expected Patterns:
WITHOUT Post-Processing:
• Higher FP values (more false alarms)
• Similar or slightly higher TP (good detection)
• Trade-off between sensitivity and specificity

WITH Post-Processing:
• Reduced FP values (fewer false alarms)
• Maintained or slightly reduced TP
• Better balance between precision and recall
