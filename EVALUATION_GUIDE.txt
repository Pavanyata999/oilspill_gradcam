EVALUATION FEATURES
================================================================================

1. ROC CURVES
-------------
The system generates Receiver Operating Characteristic curves to visualize
model performance across all classification thresholds.

Generated File: roc_curves_comparison.png
Layout: Two-panel comparison (side-by-side)
  - Left Panel: WITHOUT post-processing (raw predictions)
  - Right Panel: WITH post-processing (refined segmentation)

Each panel shows:
  • Three curves (SegNet, DeepLabV3+, Hybrid)
  • AUC (Area Under Curve) values in legend
  • Random classifier baseline (diagonal line)
  • Grid for easy reading

Interpretation:
  • Curve closer to top-left corner = better performance
  • AUC value interpretation:
    - 0.90-1.00: Excellent
    - 0.80-0.90: Good
    - 0.70-0.80: Fair
    - 0.60-0.70: Poor
    - 0.50-0.60: Fail


2. CONFUSION MATRICES
----------------------
Detailed breakdown of model predictions vs ground truth.

Generated Files:
  • confusion_matrices_without_segmentation.png
  • confusion_matrices_with_segmentation.png

Layout: Three matrices side-by-side (one per model)
Matrix Structure:
                    Predicted
                 No Oil | Oil Spill
    Actual  No Oil  TN  |    FP
           Oil Spill FN  |    TP

Color Coding:
  • WITHOUT: Blue heatmap
  • WITH: Green heatmap
  • Darker colors = higher counts

Key Metrics from Matrix:
  • Precision = TP / (TP + FP)
  • Recall = TP / (TP + FN)
  • Accuracy = (TP + TN) / (TP + TN + FP + FN)
  • Specificity = TN / (TN + FP)


3. METRICS COMPARISON CHART
----------------------------
Visual comparison of all performance metrics across models and approaches.

Generated File: metrics_comparison_chart.png
Layout: 2×3 grid (6 subplots, one per metric)

Metrics Visualized:
  1. Accuracy - Overall correctness
  2. Precision - Correctness of positive predictions
  3. Recall - Coverage of actual positives
  4. F1-Score - Harmonic mean of precision and recall
  5. Specificity - Correctness of negative predictions
  6. ROC-AUC - Overall discriminative ability

Each subplot shows:
  • Bar chart with two bars per model
  • Blue bars: WITHOUT post-processing
  • Red bars: WITH post-processing
  • Numerical values on top of bars
  • Y-axis: Score (0 to 1)


4. METRICS REPORT
-----------------
Text file with detailed numerical results.

Generated File: metrics_report.txt
Format: Tabulated data

Contains:
  • Two main sections (with/without post-processing)
  • Three rows per section (SegNet, DeepLabV3+, Hybrid)
  • Six columns (all metrics)
  • Four decimal precision

Easy to:
  • Copy into spreadsheets
  • Include in presentations
  • Compare across experiments


================================================================================
HOW TO USE THE EVALUATION SYSTEM
================================================================================

STEP 1: TRAIN THE MODEL (if not already trained)
-------------------------------------------------
Before evaluation, you need a trained model.

Command:
  python3 train_hybrid.py

This will:
  • Train the hybrid model on the training dataset
  • Save checkpoints to: output/checkpoints/best_model.pth
  • Display training progress and metrics
  • Take several hours depending on GPU/CPU

Expected output:
  • Training loss decreasing over epochs
  • Validation metrics improving
  • Final model saved when training completes


STEP 2: GENERATE VISUAL PREDICTIONS
------------------------------------
Generate comparison visualizations with all three models.

Command:
  python3 generate_final_results.py

This will:
  • Load the trained model
  • Process test images
  • Generate 3×3 comparison grids
  • Save to: results/final_outputs/

Output for each image:
  • Original SAR image
  • Detection mask (binary)
  • Red overlay (intuitive visualization)
  • Detection percentage shown
  • Model-specific thresholds applied

Processing time:
  • ~1-2 seconds per image with GPU
  • ~5-10 seconds per image with CPU


STEP 3: RUN COMPREHENSIVE EVALUATION
-------------------------------------
Generate ROC curves, confusion matrices, and detailed metrics.

Command:
  python3 generate_evaluation_metrics.py

This will:
  • Load the trained model
  • Process all test images (36 samples)
  • Compare models with and without post-processing
  • Generate all visualizations and reports
  • Save to: results/evaluation_metrics/

Generated files:
  1. roc_curves_comparison.png
  2. confusion_matrices_without_segmentation.png
  3. confusion_matrices_with_segmentation.png
  4. metrics_comparison_chart.png
  5. metrics_report.txt

Processing time:
  • ~3-5 minutes with GPU
  • ~10-20 minutes with CPU


STEP 4: ANALYZE RESULTS
------------------------
Review the generated files to understand model performance.

Key Questions to Answer:

1. Which model performs best overall?
   → Check F1-Score and ROC-AUC in metrics_report.txt
   → Hybrid model typically best

2. Does post-processing help?
   → Compare blue vs red bars in metrics_comparison_chart.png
   → Typically +5-10% precision improvement

3. What types of errors occur?
   → Examine confusion matrices
   → High FP = too many false alarms
   → High FN = missing actual oil spills

4. How well does the model discriminate?
   → Check ROC curves
   → AUC > 0.90 = excellent discrimination

5. Which model for deployment?
   → Hybrid: Best overall balance
   → SegNet: Minimize false positives
   → DeepLabV3+: Minimize missed detections


================================================================================
POST-PROCESSING SEGMENTATION EXPLAINED
================================================================================

WHAT IS POST-PROCESSING?
-------------------------
Post-processing refines raw neural network predictions using image processing
techniques to produce cleaner, more accurate segmentation masks.

WHY USE POST-PROCESSING?
-------------------------
Benefits:
  • Reduces false positives (fewer false alarms)
  • Removes small noise artifacts
  • Fills small holes in detections
  • Creates spatially coherent masks
  • Improves precision by 5-10%

Trade-offs:
  • Slight reduction in recall (1-3%)
  • Additional computation time (2-3x)
  • Requires parameter tuning

TECHNIQUES USED
---------------

1. Adaptive Thresholding
   Instead of fixed 0.5 threshold, uses model-specific thresholds:
   • SegNet: 60th percentile (minimum 0.45)
   • DeepLabV3+: 95th percentile (maximum 0.95)
   • Hybrid: 92nd percentile (0.65-0.92 range)

2. Morphological Opening
   Erosion followed by dilation
   • Removes small noise and isolated pixels
   • Breaks thin connections
   • Kernel size: 3-5 pixels

3. Morphological Closing
   Dilation followed by erosion
   • Fills small holes
   • Connects nearby regions
   • Kernel size: 5-7 pixels

4. Small Object Removal
   Removes connected components below size threshold
   • SegNet: 200 pixels minimum
   • DeepLabV3+: 100 pixels minimum
   • Hybrid: 120 pixels minimum

WHEN TO USE WHICH APPROACH?
----------------------------

Use WITHOUT Post-Processing when:
  • Speed is critical
  • Raw neural network performance needed
  • Comparing different architectures fairly
  • Research and development

Use WITH Post-Processing when:
  • Deploying to production
  • False positives are costly
  • Clean masks required for analysis
  • Operational monitoring


================================================================================
MODEL COMPARISON SUMMARY
================================================================================

SEGNET
------
Characteristics:
  • Conservative detection strategy
  • Precise boundary delineation
  • Memory efficient architecture
  • VGG-16 style encoder

Strengths:
  • High precision (fewer false alarms)
  • Clean detections
  • Good for minimizing false positives

Weaknesses:
  • May miss small oil spills
  • Lower recall than other models
  • Conservative predictions

Best Use Case:
  • When false positives are very costly
  • High confidence detections needed
  • Limited computational resources


DEEPLABV3+
----------
Characteristics:
  • Aggressive detection strategy
  • Multi-scale feature processing
  • ResNet-50 backbone
  • ASPP module for context

Strengths:
  • High recall (fewer missed spills)
  • Comprehensive coverage
  • Good multi-scale detection

Weaknesses:
  • More false positives
  • Lower precision
  • Higher computational cost

Best Use Case:
  • When missing spills is very costly
  • Maximum coverage needed
  • Emergency response scenarios


HYBRID MODEL
------------
Characteristics:
  • Balanced detection strategy
  • Combines SegNet and DeepLabV3+
  • Attention-based fusion
  • Learnable weights

Strengths:
  • Best overall F1-score
  • High precision AND recall
  • Robust to variations
  • Leverages both architectures

Weaknesses:
  • Most complex architecture
  • Highest computational cost
  • Longest training time

Best Use Case:
  • Operational deployment
  • Best overall performance needed
  • Production systems

================================================================================
TROUBLESHOOTING
================================================================================

ISSUE: "No module named 'torch'"
SOLUTION: Install PyTorch
  pip install torch torchvision

ISSUE: "No module named 'cv2'"
SOLUTION: Install OpenCV
  pip install opencv-python

ISSUE: "No module named 'sklearn'"
SOLUTION: Install scikit-learn
  pip install scikit-learn

ISSUE: "No module named 'seaborn'"
SOLUTION: Install seaborn
  pip install seaborn

ISSUE: "CUDA out of memory"
SOLUTION: Reduce batch size or use CPU
  • Edit the script to reduce batch size
  • Or force CPU usage (slower but works)

ISSUE: "No test images found"
SOLUTION: Check dataset structure
  • Ensure test images in: dataset/SOS_dataset/test/palsar/sat/
  • Ensure ground truth in: dataset/SOS_dataset/test/palsar/gt/

ISSUE: "Model checkpoint not found"
SOLUTION: Train the model first
  python3 train_hybrid.py

ISSUE: "All metrics are zero"
SOLUTION: Model not trained properly
  • Verify checkpoint file exists
  • Check if training completed successfully
  • Ensure correct checkpoint path

ISSUE: "Evaluation taking too long"
SOLUTION: Reduce number of test images
  • Edit generate_evaluation_metrics.py
  • Change line: image_files[:36] to image_files[:10]
